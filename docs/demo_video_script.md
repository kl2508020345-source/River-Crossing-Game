# Demo Video Script (2–3 minutes)

## 1. Opening (0:00–0:20)
"Hi everyone. This is *River Crossing Puzzle Adventure: Fantasy Kingdom Escape*, built using Python and Pygame with an object-oriented design. The goal is to safely transport a Wizard and four fantasy entities across a river before the timer runs out."

## 2. Start Menu + Bilingual UI (0:20–0:40)
"This is the Start Menu with a modern UI. I can toggle the interface language using the **L** key. The game supports English and Malay."

## 3. Instructions Screen (0:40–1:10)
"Here are the puzzle rules: the boat carries the Wizard plus one entity. If the Dragon is left with the Baby Phoenix without the Wizard, we lose. The Magic Crystal also cannot be left with the Potion Chest unattended. I’ll go back and start the game."

## 4. Main Gameplay + Controls (1:10–2:10)
"In the main game, I use **TAB** to select an entity. **B** boards or unboards the Wizard. **E** loads or unloads the selected entity, and **ENTER** makes the boat cross—only if the Wizard is on the boat. Movement is animated smoothly, and the HUD shows time, moves, score, and best score."

"I’ll demonstrate a mistake: if I leave the Dragon with the Baby Phoenix on a bank while the Wizard is away, the game immediately detects a rule violation and shows the lose screen."

"Now I’ll restart and solve it correctly. When all entities reach the right bank, the game shows the win screen and calculates a score based on remaining time and moves." 

## 5. Multiple Levels + Best Score Save (2:10–2:40)
"This project includes multiple levels with different time limits and crossing speed. The best score is saved locally, so it persists across runs."

## 6. Closing (2:40–3:00)
"That concludes the demo. The project includes a UML diagram, a reflection report, and automated tests for the puzzle logic and timer. Thank you."
